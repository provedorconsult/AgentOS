# 10 - Retro

Objetivo: capturar aprendizado e melhorar o sistema apos uma entrega.

## Entrada

- `docs/REVIEW.md`
- Git diff
- Falhas encontradas
- Validacoes executadas
- Feedback do usuario

## Passos

1. Identifique o que funcionou.
2. Identifique o que falhou.
3. Verifique se a falha e recorrente.
4. Decida se deve atualizar rule, skill, workflow, template ou documentacao.
5. Crie uma task pequena para melhoria, se necessario.
6. Registre a licao.

## Saida

- Licoes registradas.
- Melhorias propostas.
- Docs, skills ou rules atualizados quando apropriado.

## Guardrails

- Nao fazer refactor amplo.
- Nao alterar politica global por caso isolado.
- Nao mascarar falhas de validacao.
