# Agent Layers

AgentOS organizes agents by operating layer while preserving the existing flat agent files for compatibility.

## Layers

- Engineering: technical planning, implementation, verification and release.
- Business: product framing and project initiation.
- Operations: adaptation, deployment and observability.
- Knowledge: continuous improvement, environment curation and future memory work.

## Current Mapping

- Engineering: `orchestrator`, `technical-planner`, `explorer`, `architect`, `implementer`, `qa`, `reviewer`, `security`, `release`, `deploy`, `observability`, `continuous-improvement`, `environment-curator`.
- Business: `product-planner`, `project-initiator`.
- Operations: `project-adapter`, `deploy`, `observability`.
- Knowledge: `continuous-improvement`, `environment-curator`, future memory curator.

## Rule

Do not duplicate or move existing agent files without a migration task. Layer folders document ownership and planned growth.
